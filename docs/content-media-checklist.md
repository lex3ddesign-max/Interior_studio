# AVENOR — карта материалов для следующего наполнения

Эта техкарта нужна, чтобы заменить текущие демонстрационные изображения на реальные материалы без переделки структуры сайта.

## Портрет / страница «О нас»

- Файл: `public/images/about/founder.jpg`
- Формат: JPG или WebP, вертикальный кадр.
- Размер: желательно от 1600×2000 px.
- Настроение: спокойный dark cinematic, мягкий боковой свет, без офисной суеты и яркого фона.
- Что важно: лицо/силуэт, рабочая атмосфера, материалы, монитор с 3D-сценой или архитектурные детали.

Пока реального фото нет, сайт использует атмосферный визуальный кадр из текущей библиотеки. В интерфейсе больше нет текста-заглушки про будущую замену.

## Кейсы

Для каждого полноценного кейса лучше подготовить минимум 8 изображений:

1. Hero / главный кадр.
2. Общий ракурс пространства.
3. Второй общий ракурс с другой точки.
4. Средний план основной зоны.
5. Деталь материалов.
6. Деталь света / отражений.
7. Альтернативный сценарий или вечерний кадр.
8. Финальный атмосферный кадр.

Рекомендуемая структура:

```text
public/images/cases/private-residence/01.jpg
public/images/cases/private-residence/02.jpg
...
public/images/cases/private-residence/08.jpg
```

Аналогично:

- `public/images/cases/coastal-villa/`
- `public/images/cases/warm-minimalism-apartment/`
- `public/images/cases/boutique-commercial-space/`
- `public/images/cases/evening-house-exterior/`

После загрузки реальных файлов нужно будет обновить массивы `gallery` и `coverImage` в `data/cases.ts`.

## SEO / превью

- Главный OG-файл: `public/images/og/avenor-main.jpg`
- Размер: 1200×630 px.
- Лучше использовать кадр с темной зоной под логотип/текст, чтобы превью в Telegram/VK выглядело дорого и спокойно.

## Контакты и соцсети

Перед финальным запуском заменить временные контактные значения на реальные:

- email;
- Telegram;
- WhatsApp;
- VK;
- ссылка на портфолио/материалы, если будет отдельное облако.

Основные точки подключения медиа собраны в `data/media.ts`.
